## Prerequisites

- A Google account with Gmail
- Google OAuth credentials ([setup guide](/docs/google-oauth-setup))

## Option 1: One-Line Install (Recommended)

The fastest way to get started.

<Tabs items={['macOS', 'Linux', 'Windows']}>
  <Tab value="macOS">
    ```bash
    curl -fsSL https://inboxorcist.com/install.sh | bash
    ```

    This installs to `~/.inboxorcist`. The installer will ask if you want to use SQLite (default) or PostgreSQL.

    Then start:
    ```bash
    ~/.inboxorcist/inboxorcist
    ```

    Or add to your PATH:
    ```bash
    echo 'export PATH="$HOME/.inboxorcist:$PATH"' >> ~/.zshrc
    source ~/.zshrc
    inboxorcist
    ```
  </Tab>
  <Tab value="Linux">
    ```bash
    curl -fsSL https://inboxorcist.com/install.sh | bash
    ```

    This installs to `~/.local/share/inboxorcist`. The installer will ask if you want to use SQLite (default) or PostgreSQL.

    Then start:
    ```bash
    ~/.local/share/inboxorcist/inboxorcist
    ```

    Or add to your PATH:
    ```bash
    echo 'export PATH="$HOME/.local/share/inboxorcist:$PATH"' >> ~/.bashrc
    source ~/.bashrc
    inboxorcist
    ```
  </Tab>
  <Tab value="Windows">
    Open **PowerShell** and run:

    ```powershell
    irm inboxorcist.com/install.ps1 | iex
    ```

    This installs to `%LOCALAPPDATA%\Inboxorcist`. The installer will ask if you want to use SQLite (default) or PostgreSQL.

    Then start:
    ```powershell
    & "$env:LOCALAPPDATA\Inboxorcist\inboxorcist.exe"
    ```

    Or add to your PATH:
    ```powershell
    [Environment]::SetEnvironmentVariable("PATH", $env:PATH + ";$env:LOCALAPPDATA\Inboxorcist", "User")
    # Restart PowerShell, then:
    inboxorcist
    ```
  </Tab>
</Tabs>

<Callout type="info">
  On first run, the browser opens automatically to the setup page. Secure secrets are generated automatically.
</Callout>

## Option 2: Manual Download

Download the latest release for your platform from [GitHub Releases](https://github.com/inboxorcist/inboxorcist/releases).

<Tabs items={['macOS', 'Linux', 'Windows']}>
  <Tab value="macOS">
    **Apple Silicon (M1/M2/M3):**
    ```bash
    curl -LO https://github.com/inboxorcist/inboxorcist/releases/latest/download/inboxorcist-darwin-arm64.tar.gz
    tar -xzf inboxorcist-darwin-arm64.tar.gz
    cd inboxorcist
    ./inboxorcist
    ```

    **Intel Mac:**
    ```bash
    curl -LO https://github.com/inboxorcist/inboxorcist/releases/latest/download/inboxorcist-darwin-x64.tar.gz
    tar -xzf inboxorcist-darwin-x64.tar.gz
    cd inboxorcist
    ./inboxorcist
    ```
  </Tab>
  <Tab value="Linux">
    **x64 (most servers):**
    ```bash
    curl -LO https://github.com/inboxorcist/inboxorcist/releases/latest/download/inboxorcist-linux-x64.tar.gz
    tar -xzf inboxorcist-linux-x64.tar.gz
    cd inboxorcist
    ./inboxorcist
    ```

    **ARM64 (Raspberry Pi, AWS Graviton):**
    ```bash
    curl -LO https://github.com/inboxorcist/inboxorcist/releases/latest/download/inboxorcist-linux-arm64.tar.gz
    tar -xzf inboxorcist-linux-arm64.tar.gz
    cd inboxorcist
    ./inboxorcist
    ```
  </Tab>
  <Tab value="Windows">
    1. Download [`inboxorcist-windows-x64.zip`](https://github.com/inboxorcist/inboxorcist/releases/latest/download/inboxorcist-windows-x64.zip)
    2. Extract the ZIP file
    3. Open the extracted folder
    4. Double-click `inboxorcist.exe`
  </Tab>
</Tabs>

## Option 3: Docker

If you prefer Docker:

```bash
# Create a directory
mkdir inboxorcist && cd inboxorcist

# Create docker-compose.yml with pre-built image
cat > docker-compose.yml << 'EOF'
services:
  inboxorcist:
    image: ghcr.io/inboxorcist/inboxorcist:latest
    ports:
      - "6616:6616"
    environment:
      - JWT_SECRET=${JWT_SECRET}
      - ENCRYPTION_KEY=${ENCRYPTION_KEY}
    volumes:
      - inboxorcist-data:/usr/src/app/data

volumes:
  inboxorcist-data:
EOF

# Generate secrets
cat > .env << EOF
JWT_SECRET=$(openssl rand -base64 32)
ENCRYPTION_KEY=$(openssl rand -hex 32)
EOF

# Start Inboxorcist
docker compose up -d
```

Open http://localhost:6616/setup to configure Google OAuth via the web UI.

## Configure Google OAuth

On first run, you'll be redirected to the setup page to configure Google OAuth.

<Callout type="info">
  See the [Google OAuth Setup](/docs/google-oauth-setup) guide for step-by-step instructions on creating OAuth credentials.
</Callout>

**Configuration options:**
- **Web UI (Recommended)** - Enter credentials at `/setup` on first run
- **Environment variables** - Set `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` in `.env`

## What Gets Started

| Method | URL | Database | Queue |
| ------ | --- | -------- | ----- |
| Binary | http://localhost:6616 | SQLite (default) or PostgreSQL | In-memory |
| Docker | http://localhost:6616 | PostgreSQL | In-memory |

## Connect Your Gmail

1. Open http://localhost:6616 in your browser
2. Click **Connect Gmail**
3. Sign in with your Google account
4. Grant the requested permissions
5. Start cleaning your inbox!

## Stopping the Application

**Binary:**
```bash
# Press Ctrl+C in the terminal
```

**Docker:**
```bash
# Stop containers (keeps data)
docker compose down

# Stop and remove all data
docker compose down -v
```
