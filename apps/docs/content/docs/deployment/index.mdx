Inboxorcist can be deployed in multiple ways depending on your needs. All methods support the full feature set.

## Comparison

| Method | Difficulty | Cost | Best For |
| ------ | ---------- | ---- | -------- |
| [Single Binary](/docs/deployment/binary) | Easiest | Free (self-hosted) | Personal use, zero dependencies |
| [Docker](/docs/deployment/docker) | Easy | Free (self-hosted) | Personal use, full control |
| [Railway](/docs/deployment/railway) | Very Easy | Free tier available | Quick deployment, no infrastructure |
| [Render](/docs/deployment/render) | Very Easy | Free tier available | Automatic deploys, easy scaling |
| [Fly.io](/docs/deployment/fly-io) | Easy | Free tier available | Global distribution, edge compute |
| [DigitalOcean](/docs/deployment/digitalocean) | Easy | From $5/mo | Reliable PaaS, integrated databases |
| [Custom VPS](/docs/deployment/vps) | Moderate | Varies | Maximum control, existing infrastructure |

## Prerequisites for All Methods

Before deploying, you'll need:

1. **Google OAuth Credentials** - [Setup guide](/docs/google-oauth-setup)
2. **Security Secrets** - Generated by the setup script or manually:

```bash
# Generate JWT_SECRET (32+ characters)
openssl rand -base64 32

# Generate ENCRYPTION_KEY (64 hex characters)
openssl rand -hex 32
```

## Architecture Options

### Default Architecture

The default deployment runs a single server with bundled PostgreSQL:

```
┌─────────────────────────────────┐
│         Docker Container        │
│  ┌─────────────────────────┐    │
│  │   Inboxorcist Server    │    │
│  │   (API + SPA on :6616)  │    │
│  └────────────┬────────────┘    │
└───────────────┼─────────────────┘
                │
                ▼
       ┌─────────────────┐
       │    PostgreSQL   │
       └─────────────────┘
```

## Quick Start Commands

```bash
# Default setup (bundled PostgreSQL)
docker compose up -d

# With external PostgreSQL (set DATABASE_URL in .env)
docker compose --profile external-db up -d
```
