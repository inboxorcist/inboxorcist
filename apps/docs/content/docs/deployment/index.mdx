Inboxorcist can be deployed in multiple ways depending on your needs. All methods support the full feature set.

## Comparison

| Method | Difficulty | Database | Best For |
| ------ | ---------- | -------- | -------- |
| [Single Binary](/docs/deployment/binary) | Easiest | SQLite or PostgreSQL | Personal use, VPS, cloud servers (Recommended) |
| [Docker](/docs/deployment/docker) | Easy | PostgreSQL | Users who prefer containers |
| [Railway](/docs/deployment/railway) | Very Easy | PostgreSQL | Quick cloud deployment |
| [Render](/docs/deployment/render) | Very Easy | PostgreSQL | Automatic deploys from Git |
| [Fly.io](/docs/deployment/fly-io) | Easy | PostgreSQL | Global edge distribution |
| [DigitalOcean](/docs/deployment/digitalocean) | Easy | PostgreSQL | Managed PaaS |
| [Custom VPS](/docs/deployment/vps) | Easy | SQLite or PostgreSQL | Maximum control |

<Callout type="info">
  **Recommended:** The single binary is the simplest way to deploy Inboxorcist. It supports both SQLite and PostgreSQL, requires zero dependencies, and works on any VPS or cloud server.
</Callout>

## Prerequisites for All Methods

Before deploying, you'll need:

1. **Google OAuth Credentials** - [Setup guide](/docs/google-oauth-setup)
2. **Security Secrets** - Generated by the setup script or manually:

```bash
# Generate JWT_SECRET (32+ characters)
openssl rand -base64 32

# Generate ENCRYPTION_KEY (64 hex characters)
openssl rand -hex 32
```

## Architecture Options

### Binary (Recommended)

The binary runs as a single process with your choice of database:

```
┌─────────────────────────────┐
│   Inboxorcist Binary        │
│   (API + SPA on :6616)      │
└─────────────┬───────────────┘
              │
              ▼
    ┌─────────────────┐
    │ SQLite (local)  │  ← Default: zero setup
    │       OR        │
    │ PostgreSQL      │  ← Cloud deployments
    └─────────────────┘
```

### Docker

Docker deployment with bundled PostgreSQL:

```
┌─────────────────────────────────┐
│         Docker Container        │
│  ┌─────────────────────────┐    │
│  │   Inboxorcist Server    │    │
│  │   (API + SPA on :6616)  │    │
│  └────────────┬────────────┘    │
└───────────────┼─────────────────┘
                │
                ▼
       ┌─────────────────┐
       │    PostgreSQL   │
       └─────────────────┘
```

## Quick Start Commands

**Binary (Recommended):**
```bash
curl -fsSL https://inboxorcist.com/install.sh | bash
```

**Docker:**
```bash
docker compose up -d
```
